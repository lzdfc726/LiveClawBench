# Dashboard KPI Formula Specification

## Valid Field Names (v2, current)

| Field Name | Description |
|---|---|
| `budget_amount` | Department budget allocation for the month |
| `actual_expense_amount` | Actual expenses incurred by the department |
| `revenue_amount` | Revenue generated by the department |

## Deprecated Field Names (v1, removed 2025-Q3)

The following field names are **no longer supported** and will cause formula validation errors if used:

| Deprecated Name | Current Replacement |
|---|---|
| `total_expenses` | `actual_expense_amount` |
| `budget_deviation` | `budget_amount` |
| `gross_revenue` | `revenue_amount` |

## Formula Operators

| Operator | Arguments | Description |
|---|---|---|
| `field` | `name` (string) | Reference a numeric field value from department records |
| `const` | `value` (number) | A constant numeric value |
| `add` | `left`, `right` (FormulaNode) | Addition |
| `subtract` | `left`, `right` (FormulaNode) | Subtraction |
| `multiply` | `left`, `right` (FormulaNode) | Multiplication |
| `divide` | `left`, `right` (FormulaNode) | Division (returns null if divisor is 0) |
| `sum` | `args` (FormulaNode array) | Sum of all argument values |

## Constraints

- Maximum formula nesting depth: **5 levels**
- Only v2 field names are accepted in formula validation
- All constant values must be finite numbers (no `Infinity` or `NaN`)

## Historical Reference (2025-Q3 Migration)

Prior to the v2 migration in 2025-Q3, the company used a simplified KPI formula expressed as:

> **Net Adjusted Performance Index (NAPI)** =
> `Revenue - (Total Expenses x 0.10) + ((Budget Deviation - Total Expenses) x 0.05)`

**Important**: This historical formula definition uses deprecated v1 field names. When implementing this formula in the current system, all field references must be updated to their v2 equivalents (see table above).

The intended behavior is: for each department in each month, compute a custom performance metric that accounts for revenue, expenses, and budget utilization.

## Active Configuration

The current dashboard configuration is maintained via the admin user's settings:
- **Date range**: Calendar year 2026 (2026-01-01 through 2026-12-31)
- **Department weights**: Engineering (1.0), Sales (1.2), Marketing (0.8); all others default to 1.0

### Department Weights

| Department | Weight |
|---|---|
| Engineering | 1.0 |
| Sales | 1.2 |
| Marketing | 0.8 |
| HR | 1.0 |
| Finance | 1.0 |
| Operations | 1.0 |

Departments not listed in the weight configuration use a default weight of 1.0.

## Validation Rules

When updating the dashboard configuration via the API, the system validates:
1. Date range is valid (start <= end)
2. `formula_json` is valid JSON and passes structural validation
3. `department_weight_json` is valid JSON with numeric values

If validation fails, the dashboard falls back to default metrics (unweighted revenue and expense sums).
