# Software Patent Application Guide for Independent Developers

## Introduction

This guide provides step-by-step instructions for filing software patent applications with the USPTO, EPO, and CNIPA.

## Patentability Requirements

### 1. Patentable Subject Matter
Software must be tied to a specific technical application or improvement. Abstract ideas alone are not patentable following Alice Corp v. CLS Bank.

### 2. Novelty
The invention must be new and not disclosed in prior art.

### 3. Non-Obviousness
The invention must not be obvious to a person having ordinary skill in the art (PHOSITA).

### 4. Utility
The invention must have a specific, substantial, and credible utility.

## Filing Process

### USPTO (United States)
1. Prepare specification, claims, drawings, abstract
2. File via EFS-Web or Patent Center
3. Pay filing fees ($300-$1,720 depending on entity size)
4. Respond to Office Actions

### EPO (Europe)
1. File European patent application
2. Search report and written opinion
3. Substantive examination
4. Grant and validation in designated states

### CNIPA (China)
1. File in Chinese
2. Formality examination
3. Substantive examination (upon request)
4. Grant and publication

## Maintenance Fees
Patents require periodic maintenance fees to remain in force. Failure to pay results in expiration.

## Note
This guide focuses on software patent protection. Copyright protection for software code is a separate legal mechanism governed by different rules.
