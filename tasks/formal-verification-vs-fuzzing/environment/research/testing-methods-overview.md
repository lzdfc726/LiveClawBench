# Overview of Software Testing Methods

## Introduction

Software testing encompasses a wide range of techniques aimed at verifying that software behaves as expected. This document provides a high-level overview of common testing approaches.

## Categories of Testing

### 1. Unit Testing
Tests individual functions or modules in isolation. Typically automated and fast.

### 2. Integration Testing
Tests interactions between modules or services.

### 3. System Testing
Tests the complete system against requirements.

### 4. Acceptance Testing
Validates that the system meets user needs.

## Manual vs. Automated Testing

- **Manual testing**: Human testers execute test cases
- **Automated testing**: Scripts and tools execute tests automatically

## Static vs. Dynamic Analysis

- **Static analysis**: Examines code without execution
- **Dynamic analysis**: Examines code during execution

## Note

This document provides general background on software testing. Formal verification and fuzzing are specialized techniques within this landscape that require dedicated investigation.
